# Google カレンダー連携セットアップ手順

「来店予約」「オンライン相談予約」フォームが送信されたら、
`info.deen22@gmail.com` の Google カレンダーに「（仮）予約」として予定を自動作成します。

## セットアップ（5分）

1. **info.deen22@gmail.com で Google にログイン**した状態で
   <https://script.google.com/> を開く

2. 「**新しいプロジェクト**」をクリック

3. 左側のエディタに表示された `function myFunction()` の中身を全削除し、
   `Code.gs` の内容をそのままコピペして保存（Ctrl+S）。
   プロジェクト名は例: `Deen予約カレンダー`

4. 画面右上の「**デプロイ**」→「**新しいデプロイ**」を選択

5. 歯車アイコン → 「**ウェブアプリ**」を選択

6. 設定:
   - **説明**: 予約カレンダー連携
   - **次のユーザーとして実行**: 自分（info.deen22@gmail.com）
   - **アクセスできるユーザー**: **全員**
   - 「デプロイ」ボタン

7. 初回のみ「承認が必要」と出るので「アクセスを承認」→
   Google アカウントを選択 →「詳細」→「（プロジェクト名）に移動（安全ではないページ）」→
   「許可」で Calendar 権限を付与

8. 発行された **「ウェブアプリ URL」**（`https://script.google.com/macros/s/xxxxxxxxxxxxxxx/exec` の形式）をコピー

9. `reserve.html` と `online.html` の以下の定数を貼り替える:
   ```js
   const GAS_ENDPOINT = 'ここにコピーしたURL';
   ```

## 動作確認

- `reserve.html` の予約フォームを送信
- `info.deen22@gmail.com` の Google カレンダーに
  「（仮）来店予約 - お名前」というイベントが第1希望の日時で作成される
- 空き時間表示（透明予定）になっているので、本予約が決まったら
  タイトルの「（仮）」を外して通常予定に切り替える運用を想定

## トラブルシューティング

- **予定が作成されない**: GAS エディタで `test_` 関数を手動実行し、
  「実行ログ」にエラーが出ていないか確認
- **権限エラー**: 再度「デプロイ」→ 既存のデプロイを編集して再承認
- **CORS エラー（コンソールに出るが動く）**: ブラウザからは `mode: 'no-cors'` で送信しているので、
  成功/失敗は読み取れないが、GAS 側では正しく受信・処理される
